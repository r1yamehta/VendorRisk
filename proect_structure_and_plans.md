This document is a comprehensive Technical Blueprint for the Python Backend refactor of VendorRisk AI. It is designed to be provided to an AI Agent to execute the transition from mock-data logic to a legitimate Comparative Risk Framework.🛡️ VendorRisk AI: Backend Refactor Specification1. Project Overview & Problem StatementCurrent vendor risk assessments are static and lack context. VendorRisk AI addresses the "Visibility Gap" by automating tech-stack discovery and comparing a vendor’s security, financial, and operational data against Industry Standard Benchmarks.The Core Logic (The Shift)Old Logic: Arbitrary point addition (e.g., +10 for a bug).New Logic (Comparability Factor): We define a Standard Risk Budget (e.g., 80). We calculate vendor "Penalty Points" based on real API data. The final output is a ratio: Vendor Points / Standard Value.Visualization: A Heat-Bar (Green $\rightarrow$ Yellow $\rightarrow$ Red) based on how much of the "Risk Budget" is consumed.2. Technical StackEnvironment: Python 3.12 managed by uv (Astral).Framework: Flask (Python) + Node.js (Orchestrator).API Sources: Wappalyzer (Tech Discovery), OSV.dev (Vulnerabilities), yfinance (Financials), NewsAPI (Reputation), HIBP (Dark Web).3. Module Specifications (The "Six Pillars")A. logic/standards.pyGoal: Define the "Industry Safe Zone."Implementation: Define a class RiskBenchmarks.Values: * STD_MAX_CVES = 10STD_UPTIME = 99.9RISK_BUDGET = 80 (The comparative anchor).B. logic/cyber_engine.pyGoal: Software Composition Analysis (SCA).Task 1: Use Wappalyzer API to identify tech stack and versions from a URL.Task 2: Pass software names/versions to OSV.dev (No API key needed).Logic: Every vulnerability found adds "Penalty Points" based on severity.C. logic/scoring_engine.pyGoal: The Analytical Brain.Task: Receive raw scores from all other logic modules.Calculation: 1. Sum all vendor penalty points.2. Ratio = Total_Points / standards.RISK_BUDGET.Output: Return JSON containing: fill_percent (0-100), color (hex code), and status_text.D. logic/financial.pyGoal: Verify business lifespan.Task: Use yfinance or OpenCorporates to fetch registration dates.Logic: If Years_Active < standards.STD_YEARS, add penalty points.E. logic/operations.pyGoal: Measure operational maturity.Task: Scrape /.well-known/security.txt.Logic: Presence of a security policy = 0 penalty. Absence = High penalty.F. app.pyGoal: API Orchestration.Route: /analyze?domain=vendor.comLogic: Calls all engines in parallel (or sequence), sends results to ScoringEngine, and returns the final Comparative Risk Report.5.
